# Reopening conditions

The project can reopen only with one of the following:

1. authentic matching firmware/debug material with the required O5F contract;
2. an authorized matched O5F 11700K/11900K pre-memory capture;
3. an authentic Intel/Lenovo debug build that exposes the generated conversion
   or runtime registry;
4. primary-source and firmware proof of safe recovery from a valid bad
   pre-memory state without external SPI; or
5. a documented trusted read-only interface to the exact required state.

Every submission needs package/capture SHA-256, BIOS and hardware identity,
capture stage, raw output, and an independently reproducible method. See the
templates directory for mandatory fields. A generic 11900K board, a clock
screenshot, or a patch proposal is not a reopening trigger.
